# Duplicate Request Analysis: Enach (Mandate) Component - Razorpay + Digio Only

## Critical Issues Causing Duplicate Mandate Requests

### 1. **MULTIPLE CONCURRENT MANDATE REQUESTS** ❌ CRITICAL

**File:** [customer-ui/src/app/bitly/enach/enach.component.ts](customer-ui/src/app/bitly/enach/enach.component.ts#L233)

Focus is on **Razorpay** (default) and **Digio** mandate flows only.

#### Current Setup Flow
```typescript
async setup(mandateType){
    if(!this.crd.status){
      return false;
    }
    
    if (this.loanDetail.eMandatePartner == "DIGIO") {
      this.loanService.digioPaymentMeta(this.tokenId, mandateType).subscribe(...);
    } else {
      // Razorpay (default)
      this.loanService.paymentMeta(this.tokenId, mandateType).subscribe(...);
    }
    
    this.showPage = false;  // ← Only flag, no request tracking
}
```

#### Problem
- **No request deduplication guard**
- User can click "Setup" button multiple times
- Each click triggers a new mandate creation request
- **Result:** Multiple mandates created in database

### Fix #6: Add Callback Deduplication for Razorpay (MEDIUM)

```typescript
private razorpayCallbackFired = false;  // ✅ NEW: Add flag

public options: any = {
    // ...
    handler: this.paymentResponseHandler.bind(this),
    modal: {
      ondismiss: (() => {
        this.onEmandateDismiss();
      })
    }

paymentResponseHandler(response: any) {
    // ✅ NEW: Prevent duplicate callbacks
    if (this.razorpayCallbackFired) {
        return;
    }
    this.razorpayCallbackFired = true;
    
    if (response.razorpay_payment_id) {
        this.loanService.eNach(this.tokenId, {
            razorpayPaymentId: response.razorpay_payment_id
        }).subscribe(
            (res: any) => {
                this.toasterService.pop('success', '', 'Mandate setup successful');
                this.router.navigateByUrl(...);
            },
            (error) => {
                this.razorpayCallbackFired = false;  // ✅ Reset on error
                this.toasterService.pop('error', '', 'Failed to finalize mandate');
            }
        );
    }
}
```

## Root Cause of Duplicate Mandates (Razorpay + Digio Only)

### Scenario: Customer clicks Setup and mandate is created twice

```
Timeline:
─────────────────────────────────────────────────────────────

1. Component ngOnInit loads
   └─> Backend returns RazorpayMandateDetails (if exists)
   └─> ✗ Component ignores mandate status
   └─> Shows Setup button even if mandate exists

2. Customer clicks Setup button
   └─> ✗ No deduplication guard
   └─> ✗ mandateRequestInProgress flag doesn't exist
   └─> Request #1 sent to backend
   └─> Popup opens (Razorpay/Digio)

3. [FAST CLICK - Before popup fully loads]
   └─> Customer clicks Setup again
   └─> ✗ No flag preventing second call
   └─> Request #2 sent to backend
   └─> Second request queued

4. Backend processes Request #1
   └─> ✗ No idempotency check
   └─> Creates mandate #1 in DB

5. Backend processes Request #2
   └─> ✗ Still no validation that mandate exists
   └─> Creates mandate #2 in DB ❌❌

6. Callback fires:
   - Razorpay callback → updateENach called
   - Digio callback → digioMandateStatus fetched → paymentResponseHandler called
   └─> Only processes one mandate, but two exist in DB
```
